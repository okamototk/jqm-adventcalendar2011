Addenpa Calendar 2011

第13話: 参戦方法
(2011/12/13)

相変わらず、翼君、岬君ツートップの如き、2名で進めている当Advent Calendarであるが、実は参戦したいのだが、どうやって参戦したいか分からないというコメントを多数の諸兄から頂いた。ネタも切れつつあることだし、今日は参戦方法について整理していきたいと思う。

ATNDにエントリー

まず、ATNDにエントリーして欲しい。ATNDのページの右上のログインしてくださいをクリックし、twiter/googleアカウントなどでログインし、参加するボタンを押して参戦する旨をコメントを入力すれば完了だ。何日の記事を担当するかも書いてくれると助かる。

執筆・公開

執筆はブログに執筆してもいいし、jQuery Mobileで執筆してもいい。執筆が終わったら、ATNDの下の方のコメント欄に執筆が完了した旨とURLを報告すれば公開は完了だ。

作風は基本上から目線の変態的な記事が多いが、ネタが切れてきたので、正直、

なんでもいい

。上から目線の記事にしたい場合は、適当に作成した文書の末尾を「だ・である」に書き換えて、「あなた・読者」を「愚民」に置換すれば簡単に作成できる。是非、会社で作成する文書で一度試してもらいたい。 jQuery Mobile版のjQuery Mobile Advent Calendar 2011はGitHubで管理してる。以下、GitHubを利用して執筆する方法を紹介しよう。これは、jQuery Mobileを使わなくても参考になる情報なので、是非お読み頂きたい。既にGitHubの使い方を修得している貴兄、貴女にとっては不要な情報なので、今更そんなこと書くなよとクレームをつける前に速やかに立ち去るのがよかろう。

準備

最初にアカウントの作成とフォークをしよう。ちなみにフォークはあってもスプーンはない。助手とは逆なのである。それはさておき、アカウントの作成はここから行う。アカウントができたら、GitHubの右へのページの「Login」からログインをしよう。

次に、GitHub上にあるjquery mobile advent calendar2011のリポジトリを開く。右上のForkと書かれたボタン(灰色のさくらんぼみたいなアイコンがあるボタンがそれだ！！)をクリックしてフォークしよう。すると、自分のリポジトリとしてフォークされる。画面上部の猫みたいなアイコンと自分のアカウント名がある部分をクリックすると、そなたのページが表示されるが、Repositoriesにjqm-adventcalendar2011というリポジトリが作成されているはずだ。

チェックアウト・編集・コミット

チェックアウトは次のようにしよう。 

$ git clone https://xxxxx@github.com/xxxxx/jqm-adventcalendar2011
$ cd jqm-adventcalendar2011
$ git branch -a
...
 remotes/origin/gh-pages
...
$ git checkout gh-pages  # ブランチ変更
ちなみにxxxxxxは18禁の表現に使う伏字ではなく貴方のアカウント名だ。それが不満であれば、色々と妄想してもいいだろう。チェックアウトするときは、自分のアカウント名を入れるのを忘れないようにしよう。後でアカウント設定が必要になり面倒だから。余談だが、GitHubは、gh-pagesブランチでGitHubのWebページを管理している。gh-pagesにコミット・プッシュすれば自動的にWebページに反映されるのだ。jqm201112xx.htmlというファイルがあるので、自分が担当したい日の名前に適用にコピーして編集しよう。index.htmlを書き換えて、自分の記事へのリンクも追加するのも忘れずに。編集が完了したら、コミットしてリポジトリへ反映させよう。 
$ git add jqm201112xx.html(貴方が追加したファイル)
$ git add -p  #  (変更点を確認)
$ git commit
$ git push
しばらくすると、GitHubからWebページのビルドに成功した旨の通知が届きhttp:/xxxxx.github.com/jqm-adventcalendar2011/にアクセスするとWebページにアクセスできるようになっているはずだ。くどいようだが、xxxxxは18禁の表現に使う伏字ではなく貴方のアカウント名だ。ちなみに、cygwinのgitではプッシュできないことがあるかも知れないので、Windowsを利用している場合、MSysGitを利用することをお薦めする。

プルリクエスト

さて、この状態では、貴方のリポジトリ上でWebページを公開しているだけだ。このリポジトリを私のリポジトリへマージするには、プルリクエストを送ればいい。画面上部のPull Requestのボタンをクリックしよう(山の上に矢印が付いているボタンだ)。あとは、Base branchにgh-pagesをHead branchにもgh-pagesを指定して、Update Commit Rangeボタンを押せばプルリクエストの送信は完了だ。

送信されたプルリクエストは私の方で確認できるようになっており、ボタン一つで私のリポジトリにマージできる。便利な世の中になったものだ。プルリクエストが完了したのを見計らって、ATNDのページに記事を公開した旨を流したり、twitterで大々的に宣伝すればこれで完了だ。

jQuery Mobile Addenpa Calender では 執筆者を募集している。（聖闘士）聖夜までの熱い盛り上がり、貴兄にもご参加頂きたい。
